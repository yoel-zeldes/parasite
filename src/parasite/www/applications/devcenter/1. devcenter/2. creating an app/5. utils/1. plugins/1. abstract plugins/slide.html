The utils library contains basic plugin objects which can be extended in order to create a new plugin.

These live in the UtilsPlugins provider (under the utils module)

The following plugins are provided:

LimitedKidsPlugin

This plugin should be used by plugins that can't support too many kids:

canHandle is implemented by this plugin in a way that tests if the current node's number of kids exceeds the limit (by default: 1000).

If the limit is exceeded, canHandle returns false, and as stated in the plugins section, the next plugin's contents will be displayed to the user.

This plugin should be used (extended actually) when the navigation model could potentially be really large (and flat). Why? for two reasons:

  • Having too many kids isn't UX-friendly - the user can't really navigate between thousands of kids.
  • Responsiveness: if there are too many kids your plugin might make the site not responsive (or even crash), due to a too big DOM for the browser to handle.
