There are various directives we've already built for your use.
Whenever you need a directive, first look here.
If the utils library doesn't have it (and you think it should), we encourage you to write the directive in a good, readable and reusable way, and add it to the utils (so others will benefit from it as well).

para-stretch

This directive is used as an attribute.

It makes sure the element's height is adjusted according to the available space:

  • If the browser window's height gets bigger, the element will be stretched to fill in the space.
  • If the browser window's height gets smaller, the element will be shrinked so the window won't have a scrollbar.

This directive should be used by elements inside a plugin's template in two cases:

  • When an element in the template might be two tall and we fear that the window will have a scrollbar. This isn't a desired user experience, and para-stretch should be used in order to avoid it (see the para-scrollbar directive for a solution for viewing the element's content that gets hidden because of the shrink).
  • When an element can get along with any available height, and the taller it gets the better it looks to the user (the circles navigation plugin is a good example for it).
    para-stretch makes sure your element will get as big as possible, so it looks awesome.

The available space is calculated by comparing the total height of the plugin's template to the available browser window height.
As a consequence, if the plugin contains elements which are positioned outside of the flow (e.g by using absolute positioning), they aren't considered part of the plugin's height.

Only one para-stretch directive is permitted per plugin.

Note: this directive won't work on elements such as span. Use div instead.

para-scrollbar

This directive is used as an attribute.

In coordination with the para-stretch directive, this directive is responsible for displaying a slick and beautiful scrollbar over the para-stretch element whenever its content doesn't fit to the browser window, and is shrunk.

In most cases, whenever one uses the para-stretch directive, the para-scrollbar directive should be used as well.

para-tooltip

This directive is used as an attribute.

The text used by this attribute will appear as a bootstrap tooltip whenever the user hovers over the element.

If this attribute changes (e.g. by using an angular expression), the tooltip changes accordingly.

para-multi-selection, para-master-select, para-select

These directives cooperate together in order to give a multi-selection experience.
para-master-select and para-select must be nested within a para-multi-selection.

Each para-select creates a checkbox.
The para-master-select creates a checkbox which controls all of the para-select in a two-way manner:

  • Checking / clearing the para-master-select will check /clear all of the para-select.
  • Selecting / clearing all of the para-selects will select /clear the para-master-select.
  • Selecting only a number of the para-selects will cause the para-master-select to transfer to an intermediate state.

para-toggle-button

This directive is used as an element.

It creates a toggle button with state bound to the para-state attribute.
One should pass in to the para-state an object with a state field.
The state field is two-way linked to the button's toggle state.

para-pretty-toggle

This directive is used as an element.

It creates a toggle button with state bound to ng-model.

It also supports an activated attribute:

  • If this attribute's expression evaluates to true, the button will support highliting visual effects (triggered in mouse hover).
  • If this attribute's expression evaluates to false, when the user hovers over the button it won't visually respond.

para-loading-button

This directive is used as an attribute on a button.

The value of this attribute should be an angular expression.
Once this expression is evaluated to true, the button will change to a loading button, meaning - it's content will change to a rotating loading icon.
Once the expression is evaluated to false, the loading animation stops, and the original content is restored.

para-pagination

This directive is used as an element.

It enables a basic pagination, e.g - one can split a big list using this directive.
Its visual appearance is a row of buttons - each represents a page. Clicking on it will change the state of the directive (the current attribute which is described later is changed). One can watch on the current page value, and display the appropriate data to the user.

The following attributes are used by the directive:

  • pages: the number of pages.
  • current: an object which should have a page field. This field is two-way linked to current page the user has chosen.
  • window: the number of buttons the pagination directive has.
    If pages > window, the most left and right buttons will have an arrow icon on them (meaning - there are more pages).

para-file-modal

This directive is used as an element.

It simply uses a template to create a bootstrap modal element with a visual UX appropriate for displaying a file contents.

It supports the following attributes:

  • title: the title of the modal.
  • path: the path of the file.

The contents of the file should be nested inside of the directive (these will be displayed in the modal's body).

In order to show the modal, one should give an id to the element, and then display the modal using the regular bootstrap way (either by using a link to it or by using the JavaScript API).

para-crop

This directive is used as an element.

It should be used whenever you've got a one line of text which is too long to be displayed.

This directive crops the text and appends ellipsis. In order to view the original text, the user can hover over the text, and a tooltip with the original text will be displayed.

It supports the following attributes:

  • text: the text to be displayed.
  • limit: the number of characters to display. Any character beyond this limit will be cropped.

para-pie

This directive is used as an element.

It displays a pie chart.
When some slice is hovered / clicked its information is displayed in the center of the pie.

It expects to find in the slices attribute an array of slice objects. Each slice has the following fields:

  • title: the title of the slice.
  • size: the size of the slice. Can be any positive number (integer or float). The sum of all of the slices' sizes don't have to add up to any specific number.
  • textPrefix, textSuffix: these are used to construct the text that will be displayed in the center of the pie: textPrefix + size + textSuffix.
