An application can support search. If it supports it, the parasite framework automatically puts a search bar (using select2) with autocomplete for the search options.

The search query contains search items. Each search item contains a filter, an operator and a value, e.g - ID = 123 is translated to filter = "ID", operator = "=", value = "123"

The user can search for multiple search items in his query.

Parasite takes care for a lot of things for you - including the GUI, the client side code, and the server side code.
The flow of searching is as follows:

  1. The user starts typing his search query. Parasite provides autocomplete of the search items based on the search options your application provides.
  2. The user finishes typing the search query, and hits the search button.
    Parasite automatically calls the application's getNavigationModel function (with update as true) in order to construct the navigation model based on the search query.
    You should handle this query both in the client side (read more about it at the application section), and in the server side.

The user can also perform sort (if supported by the navigation plugin. The table navigation plugin for instance supports it). In order to support it, you should both support it in the client side in the application, as well as in the server side.
