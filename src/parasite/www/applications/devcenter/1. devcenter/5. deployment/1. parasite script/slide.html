The parasite script is the main script in charge for your development and deployment process.

It mainly uses the Gulp build system in order to perform its tasks.

Running parasite or parasite --help will display all of the available options.

The flow of the development process is as follows:

  1. parasite dev: you should run this in the background in order to support the development environment. If you prefer, you can use parasite dev-server, which also runs the server (if you use an IDE capable of running the server, it'll be more convenient using the IDE, as you can debug the server side code using breakpoints).
  2. parasite test: you can run the tests any time you wish.

  3. parasite deploy: deploy the master branch into the production parasite server. At the end of the deployment process, if all goes well (pulling the code in parasite, building, running tests), the deploy script is run. You should put there any code which should execute in order to update the server.
