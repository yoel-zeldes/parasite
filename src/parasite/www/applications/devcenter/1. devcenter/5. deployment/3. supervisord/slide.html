The production parasite server uses supervisord in order to keep some daemons which are essential for the parasite site up and running.

Use sudo service supervisord start in order to start it in case the server restarts (note: running sudo /etc/init.d/supervisord start won't work).

The supervisord's configuration file is located at /etc/supervisord.conf.

supervisord makes sure the following are up and running in all times:

  • parasite: this daemon runs the init_site.py script (located in the src directory). This script runs code registered by all of the applications which should run only once (and not once per httpd process) - read more about it at application initialization.

If there are problems with some daemon, you can use supervisorctl to debug them, e.g. - supervisorctl tail -3000 parasite stderr.
Many times, a good old restart will solve the problem: supervisorctl restart parasite.
