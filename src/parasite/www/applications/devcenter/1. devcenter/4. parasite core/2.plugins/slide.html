The plugins_services.js file provides the Plugins provider. This provider is used both at config time in order to create new plugins, and in run time in order to access the plugins.

config
The Plugins provider should be injected (using angular DI) to a config block, or to other providers. The provider has two properties:
  • NavigationPlugin: this "class" (using the class.js package) is the base class used to create new navigation plugins. You should refer to the plugins section in order to learn about this object.
  • ContentPlugin: this "class" is the base class used to create new content plugins.
run

Once angular enters the run phase, the Plugins provider binds each plugin configuration (defined on the application objects) to its matching plugin object. This is done by matching the plugin configuration's pluginId to the plugin's id.

Once a match is found, the plugin configuration object gets a plugin property holding the plugin, and canHandle property which is the plugin's canHandle function (unless the plugin configuration has defined the function by itself).

The Plugins provider should be injected (using angular DI) to a run block, or to other services. The provider has several properties, all of which are documented in the service code itself.
