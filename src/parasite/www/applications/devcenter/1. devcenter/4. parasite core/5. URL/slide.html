The ParasiteURL service (defined in url_services.js) is responsible for encoding the state of the site in the URL. Changing some aspect of the state of the site causes changes in the URL, and changing the URL causes changes in the state.

This service is responsible both for accessing the URL, and for setting the appropriate $watches so that everything keeps up in sync, e.g. - if the user changes the current content plugin, a $watch will be triggered which makes sure to update the URL.

ParasiteURL encodes the following state in the URL:

  • The current application's name.
  • The current navigation node's id.
  • The index of the navigation plugin the user has selected.
  • The index of the content plugin the user has selected.
  • The search query the user has entered in the search box (if the application enables search).
  • The sort the user has performed.

Although accessing and modifing the site's state is possible using the ParasiteURL service, this is not the recommended way, as it makes testing harder. A more appropriate way (whenever possible) is to access the service which is actually responsible for the state, e.g. - Changing the current navigation node should be done using the Navigation service . There are situations where using the Navigation service is not an option, e.g. - when the site loads for the first time, the URL might contain the id of the navigation node which the user wants to access, so the application writer must access the ParasiteURL service directly.

Accessing all of these is documented in the code itself.
