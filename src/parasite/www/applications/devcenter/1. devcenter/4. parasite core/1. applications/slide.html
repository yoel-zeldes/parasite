The applications_services.js file provides the Applications provider. This provider is used both at config time in order to create a new application,] and in run time in order to access the applications.

config
The Applications provider should be injected (using angular DI) to a config block, or to other providers. The provider has two properties:
  • Application: this "class" (using the class.js package) is the base class used to create new applications. You should refer to the application section in order to learn about this object.
  • register: use this function in order to register an instance of the application class you created.
run
The Applications provider should be injected (using angular DI) to a run block, or to other services. The provider has two properties:
  • applications: this function returns an Array of all of the applications registered in the config phase.
  • currentApplication: a function which either returns the application the user is using (if called with no argument), or if called with an argument - set the current application to the given application.
